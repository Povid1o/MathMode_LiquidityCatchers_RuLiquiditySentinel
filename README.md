# RU Liquidity Sentinel

Проект для кейса ПСБ: система раннего предупреждения стресса ликвидности на рублевом денежном рынке.

Идея проекта - автоматически собирать публичные финансовые данные, рассчитывать несколько сигналов стресса и объединять их в один понятный индекс ликвидности.

## Цель

Сделать прототип системы, которая показывает:

- есть ли сейчас признаки стресса ликвидности;
- какие факторы сильнее всего влияют на ситуацию;
- как менялся уровень стресса в исторические периоды;
- когда нужно обратить внимание на риск дефицита ликвидности.

## Основной результат

На выходе система должна выдавать **Liquidity Stress Index (LSI)** от 0 до 100:

- `0-40` - зеленая зона, низкий стресс;
- `40-70` - желтая зона, повышенное внимание;
- `70-100` - красная зона, высокий стресс.

Также нужен дашборд с графиками, алертами и объяснением, какие модули внесли основной вклад в итоговый индекс.

## Модули системы

Проект состоит из пяти расчетных модулей.

### М1. Усреднение обязательных резервов

Смотрим данные ЦБ по обязательным резервам банков и RUONIA. Если банки сильно накапливают остатки в конце периода усреднения, это может быть сигналом напряжения с ликвидностью.

### М2. Аукционы репо ЦБ

Анализируем спрос банков на репо ЦБ. Высокий спрос и высокие ставки могут означать, что банкам не хватает краткосрочной ликвидности.

### М3. Размещение ОФЗ

Смотрим результаты аукционов ОФЗ. Низкий спрос на ОФЗ может показывать, что у банков и инвесторов мало свободных денег.

### М4. Налоговый период и сезонность

Учитываем налоговые даты, конец месяца и конец квартала. Это важно, потому что часть оттоков ликвидности является сезонной и заранее ожидаемой.

### М5. Средства федерального казначейства

Отслеживаем движение бюджетных средств. Если деньги уходят из банковской системы на счета казначейства, это может усиливать дефицит ликвидности.

## Источники данных

Основные открытые источники:

- ЦБ РФ;
- Минфин России;
- Федеральное казначейство;
- ФНС;
- данные по ОФЗ и денежному рынку.

## Как считаем индекс

1. Собираем данные из открытых источников.
2. Чистим и приводим их к единому формату.
3. Каждый модуль считает свои признаки и флаги.
4. Числовые признаки нормализуются через MAD.
5. Все сигналы объединяются в общий индекс LSI.
6. Для каждого значения индекса показывается вклад модулей.

## Backtest

Систему нужно проверить на исторических стрессовых периодах:

- декабрь 2014;
- февраль-март 2022;
- август 2023.

Цель backtest - убедиться, что индекс растет в периоды реального напряжения на рынке.
