# М2 — Аукционы репо ЦБ

## Статус

Базовый backend pipeline готов

Модуль собирает итоги аукционов репо ЦБ и ключевую ставку, формирует базовый датасет М2 для дальнейшей аналитики и ML

Feature builder для М2 пока не добавлен. Следующий шаг — дождаться требований аналитика и ML

## Источники


## Почему есть сводные и дневные raw-файлы

Страница ЦБ по репо работает в двух режимах

Если запрашивать диапазон дат, ЦБ отдает сводную таблицу:

```text
date
auction_type
term_days
auction_time
total_deals_volume
weighted_average_rate
settlement_code
```

Если запрашивать один день, ЦБ отдает детальную карточку аукциона:

```text
demand_volume
cutoff_rate
min_rate
max_rate
limit_deals_volume
weighted_average_limit_rate
first_leg_date
second_leg_date
```

Поэтому pipeline сначала скачивает сводную таблицу, получает даты реальных аукционов, а затем скачивает дневные страницы только для этих дат

Дневные HTML сохраняются в `data/raw/repo/daily/` и игнорируются git, потому что это большой воспроизводимый кеш

## Колонки repo.csv

`repo.csv` содержит:

```text
date
auction_type
term_days
auction_time
total_deals_volume
weighted_average_rate
settlement_code
demand_volume
cutoff_rate
min_rate
max_rate
limit_deals_volume
weighted_average_limit_rate
first_leg_date
second_leg_date
cover_ratio
```

`cover_ratio` считается как:

```text
demand_volume / total_deals_volume
```

В источнике ЦБ по репо нет отдельной колонки предложения. Поэтому на backend-этапе используется спрос к объему заключенных сделок. Если аналитик выберет другую методику, это нужно будет скорректировать в `m2_feature_builder.py`

## Колонки m2_dataset.csv

`m2_dataset.csv` содержит все поля `repo.csv` и поле:

```text
key_rate
```

Ключевая ставка добавляется из отдельного источника ЦБ

## Принятые решения

`repo.csv` остается source-level датасетом по аукционам репо

`m2_dataset.csv` объединяет репо и ключевую ставку

Дневной raw-кеш не пушится, потому что он занимает много файлов и полностью воспроизводится downloader'ом

`cover_ratio` добавлен как базовое техническое поле из ТЗ, а не как ML-фича

## Ограничения

Не все исторические строки имеют дневные детальные поля. Это связано с особенностями официальных страниц ЦБ:

```text
старые аукционы могут иметь неполные данные
несостоявшиеся аукционы могут не иметь спроса или ставок
детальная карточка не всегда содержит все поля
```

Ключевая ставка доступна только с периода, покрытого источником ЦБ

## Что не делается в М2

Пока не считаются:

```text
MAD_score_cover
rate_spread_to_key_rate
Flag_Demand
rolling показатели
лаги
m2_signal
```

Эти признаки должны быть добавлены позже в `m2_feature_builder.py` после требований аналитика или ML
