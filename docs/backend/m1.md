# М1 — Усреднение обязательных резервов

## Статус

Backend pipeline готов

Модуль собирает данные ЦБ по обязательным резервам и RUONIA, формирует базовый датасет М1 и отдельный датасет признаков после аналитики

## Источники

Обязательные резервы ЦБ:

```text
https://www.cbr.ru/hd_base/RReserves/
https://www.cbr.ru/vfs/hd_base/RReserves/required_reserves_table.xlsx
```

RUONIA:

```text
https://www.cbr.ru/hd_base/ruonia/dynamics/
```

## Разделение датасетов

`m1_dataset.csv` и `m1_dataset.parquet` — базовый объединенный датасет М1

`m1_features.csv` — датасет признаков для аналитика и ML

CSV оставлен для удобного просмотра и совместимости, Parquet добавлен как более строгий формат хранения типизированного датасета

## Базовый датасет

`m1_dataset` содержит:

```text
date
averaging_period_end
actual_balances
required_reserves_avg
accounting_reserves
averaging_period_days
spread
ruonia_rate
ruonia_transactions_volume
ruonia_transactions_count
ruonia_participants_count
```

`spread` считается на parser-уровне как:

```text
actual_balances - required_reserves_avg
```

## Feature dataset

`m1_features.csv` добавлен после аналитики и содержит признаки:

```text
full_reserves
spread_relative
spread_delta
spread_ma3
reserve_load
ruonia_period_avg
ruonia_start
flag_end_of_period
rolling MAD-score поля
m1_signal
m1_signal_final
m1_reliable
```

## Принятые решения

`m1_dataset` остается source-level датасетом. В нем нет MAD-score, итогового LSI и ML-логики

`m1_features` вынесен отдельно, чтобы не смешивать базовую подготовку данных и feature engineering

RUONIA в базовом датасете присоединяется по дате начала периода усреднения

Средняя RUONIA за период усреднения считается уже в `m1_feature_builder.py`, потому что это аналитический признак

`flag_end_of_period` считается в feature builder на основе даты окончания периода усреднения

## Ограничения

Часть значений RUONIA может отсутствовать для ранних дат, если источник RUONIA не покрывает весь период обязательных резервов

`m1_signal` и `m1_signal_final` являются модульными аналитическими сигналами М1, но не итоговым LSI

## Что не делается в М1

Не считаются:

```text
LSI
SHAP
итоговая агрегация всех модулей
dashboard
LLM-комментарии
```

Эти части относятся к агрегационному слою и последующим задачам
