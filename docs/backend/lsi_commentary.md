# LSI Commentary Service

Сервис автоматических аналитических комментариев по индексу стресса ликвидности (LSI).

## Что делает модуль

`backend/src/services/lsi_commentary_service.py` реализует:

1. **`load_context(threshold_profile)`** — собирает контекст из обработанных данных
2. **`build_rule_based_commentary(context)`** — генерирует текстовый комментарий без LLM
3. **`build_llm_prompt(context, user_question)`** — формирует промпт для LLM
4. **`generate_llm_commentary(context, user_question)`** — опциональный вызов OpenAI API
5. **`answer_question(question, threshold_profile, use_llm)`** — публичный API для чата

## Rule-based fallback всегда работает

Dashboard функционирует **без LLM API**. При отсутствии `OPENAI_API_KEY`:
- `generate_llm_commentary()` возвращает rule-based текст + предупреждение
- `answer_question()` отвечает на основе правил и поиска по периодам
- Страница «Аналитик» показывает предупреждение, но не падает

Fallback-цепочка: `LLM → rule-based`. Никаких исключений не бросается.

## Как включить LLM

```bash
export OPENAI_API_KEY=sk-...
# опционально — модель по умолчанию gpt-4o-mini
export OPENAI_MODEL=gpt-4o-mini
```

Пакет `openai` импортируется лениво (внутри функции), чтобы не ломать проект при его отсутствии.
При необходимости: `pip install openai`

## Данные в контексте

Модуль читает:

| Файл | Содержимое |
|------|-----------|
| `data/processed/final_ml_dataset.parquet` | Все признаки M1–M5, флаги событий |
| `data/processed/lsi_scores.parquet` | LSI Local / Global, вклады модулей |
| `data/processed/lsi_backtest_scores.parquet` | Исторические LSI (для сводки) |

В промпт передаются:
- дата последних данных
- LSI Local и LSI Global с точными значениями
- статусы по выбранному пороговому профилю
- топ-3 модулей по вкладу (Local и Global отдельно)
- флаги: M4 налоговое окно, M5 Budget Drain, M2 аномальный спрос, M3 недоспрос ОФЗ
- rule-based commentary как встроенный контекст

## Retrieval по периодам

`answer_question()` ищет ключевые слова в вопросе и подтягивает сводку по периоду:

| Ключевые слова | Период |
|---------------|--------|
| «февраль 2022», «2022-02» | 2022-02-01 — 2022-03-31 |
| «декабрь 2014», «2014-12» | 2014-11-15 — 2015-01-15 |
| «август 2023», «2023-08» | 2023-07-15 — 2023-09-15 |
| «последний», «сейчас» | последняя дата |

Если период не распознан — возвращает последние данные.

## Ограничения

- **LLM не является источником истины.** Он интерпретирует только переданный контекст.
- **Нет внешних данных.** LLM не обращается к новостям, ЦБ или другим API.
- **Дата данных ≠ сегодня.** Последняя дата в `final_ml_dataset` может отставать.
- **LSI Local ≠ LSI Global.** Local обучается на скользящем 365-дневном окне.
- **Этот модуль бонусный** — он не влияет на расчёт LSI и не меняет пороги.
- **Высокий local FP rate (~47%)** при профиле `backtest_sensitive` — нормально; требует ручного подтверждения.

## Методологические требования к LLM

LLM **должен**:
- указывать дату данных в ответе
- ссылаться на численные значения LSI
- называть модули с наибольшим вкладом
- явно писать, если данных за период нет
- различать «повышенное внимание» и «стресс ликвидности»

LLM **не должен**:
- выдумывать данные вне контекста
- утверждать «кризис точно будет»
- путать LSI Local и LSI Global
- говорить, что дата данных = сегодняшний день, если это не так
- скрывать модельную природу LSI
