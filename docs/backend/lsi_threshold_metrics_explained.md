# Профили порогов и метрики LSI

## Что такое пороговый профиль

Пороговый профиль определяет, как числовой LSI переводится в светофор:

- зелёный — норма
- жёлтый — повышенное внимание
- красный — стресс ликвидности

В проекте есть два профиля.

### conservative

```text
LSI < 40       зелёный
40 <= LSI < 70 жёлтый
LSI >= 70      красный
```

Это более осторожный профиль. Он реже даёт красный сигнал и лучше подходит для режима, где важно не перегружать казначейство ложными тревогами.

### backtest_sensitive

```text
LSI < 30       зелёный
30 <= LSI < 60 жёлтый
LSI >= 60      красный
```

Это более чувствительный профиль. Он раньше и чаще даёт красный сигнал. Такой режим подходит для бизнес-логики раннего предупреждения: лучше лишний раз поднять тревогу, чем пропустить стресс ликвидности.

## Чем отличаются Global и Local

### LSI Global

Global сравнивает текущее состояние рынка со всей историей наблюдений с 2014 года.

Экономический смысл: насколько текущий режим похож на исторически стрессовые периоды.

### LSI Local

Local сравнивает текущее состояние рынка с последним 365-дневным окном.

Экономический смысл: насколько текущий рынок выбивается из недавней локальной структуры.

Важно: Local чаще реагирует на локальные изменения и поэтому может быть более тревожным.

## Что значат метрики

### Event Recall Yellow

Доля стресс-эпизодов, где LSI дал хотя бы один жёлтый или красный сигнал.

```text
100% = все размеченные стресс-эпизоды были замечены хотя бы как "повышенное внимание"
```

### Event Recall Red

Доля стресс-эпизодов, где LSI дал хотя бы один красный сигнал.

Эта метрика показывает, насколько хорошо профиль ловит стресс именно как критический уровень.

Пример:

```text
conservative global = 33.33%
backtest_sensitive global = 100%
```

Это значит, что `conservative` поймал красным 1 из 3 стресс-эпизодов, а `backtest_sensitive` поймал красным 3 из 3.

### Lead Time Yellow

Среднее число дней от первого жёлтого сигнала до начала стресс-эпизода.

Например, `24.5` означает, что жёлтый сигнал появлялся в среднем примерно за 24–25 дней до начала стрессового окна.

### Lead Time Red

Среднее число дней от первого красного сигнала до начала стресс-эпизода.

Если значение пустое, значит в 30-дневном окне перед стресс-эпизодами красного сигнала не было.

### False Red Alerts/year

Среднее число красных сигналов в год вне размеченных стресс-эпизодов.

Это цена ложной тревожности.

Пример:

```text
conservative global = 9.62
backtest_sensitive global = 25.61
```

Это значит, что `backtest_sensitive` чаще поднимает красный сигнал вне размеченных кризисных окон.

## Как читать текущую таблицу

Главный компромисс:

```text
conservative 40/70
= меньше ложных тревог
= хуже ловит стресс красным

backtest_sensitive 30/60
= лучше ловит стресс красным
= чаще тревожит вне размеченных стресс-окон
```

Если цель — не шуметь лишний раз, лучше `conservative`.

Если цель — не пропустить стресс ликвидности, лучше `backtest_sensitive`.

## Интерактивный выбор профиля на dashboard

Dashboard позволяет аналитику **выбрать профиль прямо в интерфейсе** без изменения кода.

На странице «Обзор системы» в блоке LSI есть переключатель:

```
Пороговый профиль
● backtest_sensitive (30/60) — выше чувствительность к стрессу
○ conservative (40/70) — меньше ложных тревог
```

Выбранный профиль:
- Меняет статусы ЗЕЛЕНЫЙ / ЖЕЛТЫЙ / КРАСНЫЙ для LSI Local и Global
- Меняет горизонтальные линии на LSI-графиках
- Сохраняется в session_state и применяется также на странице «Сводные сигналы»
- **Не пересчитывает числовые значения LSI** — меняется только интерпретация

Таблица метрик в expander «Как выбрать пороговый профиль?» помогает сравнить профили
по Event Recall, Lead Time и False Alerts/year перед принятием решения.

## Практический вывод

С учётом бизнес-позиции «лучше лишний раз потревожиться, чем не заметить стресс»
production default = `backtest_sensitive`. Выбор подтверждён куратором.

Оба профиля остаются доступны и документированы как валидные:

| | conservative 40/70 | backtest_sensitive 30/60 |
|---|---|---|
| Приоритет | Меньше нагрузки на аналитика | Не пропустить кризис |
| Event Recall Red (Global) | 33% | 100% |
| False Red Alerts/year (Global) | ~9.6 | ~25.6 |

Красный сигнал при `backtest_sensitive` требует ручного подтверждения аналитиком.
