# М5: средства Казначейства и бюджетные средства в банках

## Статус

Backend pipeline М5 подготовлен: скачивает официальные данные ЦБ и Росказны, парсит source-level CSV, собирает базовый `m5_dataset.csv` и feature dataset для ML

После анализа `ml/notebooks/module_5.ipynb` добавлен отдельный feature builder. Базовый `m5_dataset.csv` остается source-level датасетом, а для обучения модели нужно использовать `m5_features.csv` или `m5_features.parquet`

## Источники

1. Бюджетные средства на счетах кредитных организаций, Банк России  
   `https://www.cbr.ru/vfs/statistics/banksector/borrowings/02_29_Budget_all.xlsx`

2. Ликвидность банковского сектора, Банк России  
   `https://www.cbr.ru/hd_base/bliquidity/`

3. Размещение средств единого казначейского счета на банковских депозитах, Росказна
   `https://roskazna.gov.ru/finansovye-operacii/razmeshchenie-sredstv-edinogo-kaznachejskogo-scheta/razmeshchenie-sredstv-edinogo-kaznachejskogo-scheta-na-bankovskih-depozitah`

Страница Росказны может не открываться из скриптов при отдельных сетевых настройках. Downloader Росказны умеет скачивать HTML-страницы архива по годам и страницам, вытаскивать из них XML-ссылки и скачивать XML. Если сеть не дает открыть `roskazna.gov.ru`, pipeline продолжает работать по уже сохраненному raw-кешу

```bash

```

Для отдельного обновления raw-файлов Росказны:

```bash
.venv/bin/python backend/src/downloaders/roskazna_treasury_deposits_downloader.py --years 2021-2026
```

Если сайт Росказны открывается в браузере, но Python падает с `CERTIFICATE_VERIFY_FAILED`, можно явно отключить проверку SSL только для downloader Росказны:


Если HTML-страницы уже сохранены локально и нужно только скачать XML из них:

```bash
.venv/bin/python backend/src/downloaders/roskazna_treasury_deposits_downloader.py --no-update-pages
```

HTML-страницы архива сохраняются в:

```bash

```

## Как собран m5_dataset.csv

`m5_dataset.csv` собирается по дневной таблице ликвидности ЦБ

К каждой дневной дате добавляется последняя известная месячная запись `currency_type = total` из `cbr_budget_funds.csv`, дата которой не позже текущей даты

По Росказне аукционы агрегируются по `auction_date` и добавляются в дневной датасет:

- `roskazna_auctions_count`
- `roskazna_max_volume_mln_rub`
- `roskazna_demand_volume_mln_rub`
- `roskazna_accepted_volume_mln_rub`
- `roskazna_settled_volume_mln_rub`
- `roskazna_weighted_average_rate`
- `roskazna_cover_ratio`
- `roskazna_accepted_ratio`
- `roskazna_settled_ratio`
- `roskazna_bidders_count`
- `roskazna_accepted_bidders_count`

Это техническое объединение источников разной частоты:

- бюджетные средства — месячная частота
- ликвидность банковского сектора — дневная частота
- депозиты Росказны — по датам аукционов

## Как собран m5_features.csv

`m5_features.csv` и `m5_features.parquet` собираются из `m5_dataset.csv`, `cbr_budget_funds.csv` и `roskazna_treasury_deposits.csv`

При сборке признаков учитываются выводы аналитического ноутбука:

- дубли Росказны удаляются по `auction_id`
- месячные бюджетные средства добавляются с лагом публикации 30 дней
- отдельно используются общие бюджетные средства и рублевые бюджетные средства
- признаки спроса на аукционах Росказны по `auction_date` сдвинуты на 1 день
- потоки Росказны считаются по `first_leg_date` и `second_leg_date`
- скользящие суммы потоков считаются за 7, 14 и 30 календарных дней назад
- смешанная ставка Росказны не переносится в признаки, потому что в XML есть аукционы с разными типами ставок

Основные группы признаков:

- лаг ликвидности банковского сектора и изменения ликвидности
- лагированные бюджетные остатки и их месячные изменения
- доля рублевых бюджетных средств в общем объеме
- лагированный спрос на аукционах Росказны
- фактические притоки и оттоки по ногам депозитных сделок
- скользящие суммы потоков Росказны
- MAD-score стресса по ликвидности ЦБ и потокам Росказны
- `Flag_Budget_Drain` для сильного чистого оттока средств Росказны
- число дней с последнего аукциона Росказны

## Важные решения

- В `m5_dataset.csv` не считаются стресс-сигналы, MAD-score, LSI и ML-признаки
- `m5_features.csv` и `m5_features.parquet` являются датасетом признаков для ML по М5
- `cbr_liquidity_stress_mad_score` считается по лагированному уровню дефицита/профицита ликвидности ЦБ; положительное значение означает ухудшение относительно rolling-медианы
- `cbr_liquidity_drain_mad_score` считается по 5-дневному изменению лагированной ликвидности; положительное значение означает отток ликвидности
- `roskazna_net_flow_stress_mad_score` считается по 14-дневному чистому потоку Росказны; положительное значение означает усиление оттока
- `Flag_Budget_Drain` равен 1, если 14-дневный чистый поток Росказны меньше или равен -300 млрд руб.
- Денежные показатели из файла бюджетных средств ЦБ оставлены в млн руб., как в источнике
- Денежные показатели таблицы ликвидности ЦБ оставлены в млрд руб., как в источнике
- Денежные показатели XML Росказны оставлены в млн руб., как в источнике
- `budget_funds_total_mln_rub` — это остаток бюджетных средств на счетах кредитных организаций, а не объем аукционов Казначейства
- `roskazna_cover_ratio` считается как `demand_volume_mln_rub / max_volume_mln_rub`
- `roskazna_accepted_ratio` считается как `accepted_volume_mln_rub / demand_volume_mln_rub`
- `roskazna_settled_ratio` считается как `settled_volume_mln_rub / max_volume_mln_rub`
- В feature builder бюджетные данные лагируются на 30 дней, чтобы не использовать месячную статистику раньше условной даты публикации
- В feature builder признаки спроса Росказны по дате аукциона сдвинуты на 1 день, чтобы снизить риск утечки same-day информации
- Parquet сохраняется через `pyarrow`, потому что стандартная библиотека Python не умеет писать Parquet
