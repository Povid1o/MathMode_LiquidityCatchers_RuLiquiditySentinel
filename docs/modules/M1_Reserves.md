# 🏦 M1 — Обязательные резервы и RUONIA

## 1. Описание модуля

**Экономический смысл.** Банки обязаны держать на корсчетах в ЦБ усреднённый за период объём резервов не ниже норматива. Внутри периода усреднения банк маневрирует остатками. Когда ликвидности в системе мало, банки **«хордят» резервы** — держат избыточную подушку на корсчетах «на всякий случай», боясь не выполнить норматив и попасть в дефицит к концу периода. Это раздувает спред «факт − норматив» и подталкивает вверх **RUONIA** (ставку овернайт по необеспеченным рублёвым межбанковским кредитам).

**За чем следим:** аномалии спреда обязательных резервов (уровень, относительный размер, нагрузка), отклонения RUONIA от нормы и **резкость движения** спреда (волатильность). M1 — крупный драйвер LSI Local (≈23% в среднем, до ~40% в отдельные даты).

> Данные публикуются **на конец периода усреднения** (≈ месяц), затем forward-fill до дневной шкалы.

## 2. Полный пул фичей (`m1_features`)

| Группа | Колонки | Смысл |
|--------|---------|-------|
| Сырьё | `actual_balances`, `required_reserves_avg`, `accounting_reserves`, `full_reserves`, `averaging_period_days` | факт/норматив резервов, длина периода |
| Спред | `spread`, `spread_relative`, `spread_delta`, `spread_ma3` | спред факт−норматив, относительный, его дельта, MA-3 |
| Нагрузка | `reserve_load` | насколько «загружены» резервы относительно норматива |
| RUONIA | `ruonia_rate`, `ruonia_period_avg`, `ruonia_start` | ставка и её средние по периоду |
| MAD-аномалии | `spread_mad_score`, `spread_relative_mad_score`, `spread_delta_mad_score`, `reserve_load_mad_score`, `ruonia_mad_score` | нормализованные аномалии каждого ряда |
| Композиты/служебные | `m1_signal`, `m1_signal_final`, `m1_reliable`, `flag_end_of_period` | готовый сигнал модуля, флаг надёжности, флаг конца периода |

## 3. Whitelist (что отобрано в LSI)

```python
M1_FEATURES = ["m1_spread_mad_score", "m1_spread_relative_mad_score",
               "m1_reserve_load_mad_score", "m1_ruonia_mad_score", "m1_spread_vol"]
```

| Признак | Почему в whitelist |
|---------|--------------------|
| `m1_spread_mad_score` | прямая аномалия «хординга» — насколько раздут спред факт−норматив |
| `m1_spread_relative_mad_score` | тот же сигнал, нормированный на масштаб резервов (устойчив к росту базы) |
| `m1_reserve_load_mad_score` | аномалия нагрузки резервов — комплементарный взгляд на тот же стресс |
| `m1_ruonia_mad_score` | ценовое подтверждение: дорожает овернайт-фондирование |
| `m1_spread_vol` | **новый признак** — резкость движения спреда (волатильность), которой не хватало |

**Почему НЕ вошли:**
- `m1_signal`, `m1_signal_final` — готовые композитные «сигналы-подсказки»: они уже агрегируют MAD-скоры, и в PCA создавали бы утечку/двойной счёт. Честный принцип: модель сама взвешивает первичные аномалии.
- `spread_delta_mad_score` (знаковый) заменён на **`spread_vol = |spread_delta_mad_score|`**: для стресса важна *величина* рывка, а не его знак.
- `flag_end_of_period`, сырые уровни (`spread`, `actual_balances`, …) — нестационарны / детерминированы, не несут самостоятельного сигнала аномалии.

## 4. Методика расчёта

### MAD-score (база для всех `*_mad_score`)

Робастная z-оценка по скользящему окну (медиана + median absolute deviation), `_mad_rolling`:

```python
med = s.rolling(win=756, min_periods=120).median()           # окно ~3 года торговых дней
mad = (s - med).abs().rolling(win, min_periods=120).median()  # MAD
score = ((s - med) / mad.clip(lower=0.05)).clip(-5, 5)        # пол MAD=0.05, клип ±5
```

Почему MAD, а не z по среднему/σ: медиана и MAD устойчивы к выбросам (кризисные точки не «портят» оценку нормы), а пол `0.05` страхует от деления на ноль в спокойные периоды.

### `spread`
```
spread = actual_balances − required_reserves_avg
```
Положительный и раздувающийся спред = банки держат избыток (хординг).

### `m1_spread_vol` (волатильность спреда)
```python
m1_spread_vol = | m1_spread_delta_mad_score |
```
Абсолютная величина аномалии **изменения** спреда: ловит резкие развороты подушки ликвидности независимо от направления.

### RUONIA
`ruonia_mad_score` — MAD-аномалия дневной RUONIA: рост ставки овернайт подтверждает удорожание рублёвого фондирования.
