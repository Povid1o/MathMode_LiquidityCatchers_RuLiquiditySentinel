# 📜 M3 — Аукционы ОФЗ (Минфин)

## 1. Описание модуля

**Экономический смысл.** Минфин занимает на внутреннем рынке через аукционы ОФЗ. Спрос на госдолг — индикатор **свободной рублёвой ликвидности** у банков и их аппетита к риску. Когда ликвидности мало или рынок нервничает:

- падает **переподписка** (`cover`) и **доля размещения** (`placement`) — спроса не хватает, **недоспрос** (Flag_Nedospros);
- Минфин вынужден давать **премию по доходности** к ключевой ставке (`yield_to_key`), чтобы продать выпуск;
- в крайнем случае аукцион **признаётся несостоявшимся** (placed = 0) — сильнейший сигнал стресса (как в феврале–марте 2022, когда размещения были приостановлены).

**За чем следим:** недоспрос/неразмещение, премия доходности к ключевой, несостоявшиеся аукционы. M3 — **крупнейший по среднему вкладу канал (≈30%)**: рынок ОФЗ быстро и ёмко отражает общий стресс. Данные **разрежены** (аукционы обычно по средам).

## 2. Полный пул фичей

**`m3_features`** (сводка по дню аукциона):

| Колонки | Смысл |
|---------|-------|
| `demand_amount`, `offered_amount`, `placed_amount` | спрос / предложение / размещено |
| `weighted_yield`, `yield_spread` | средневзв. доходность, спред к ориентиру |
| `cover_ratio` | `demand / offered` |
| `Flag_Nedospros`, `Flag_Perespros` | флаги недоспроса / переспроса |
| `MAD_score_cover`, `MAD_score_yield_spread` | календарные MAD-аномалии |

**Event-aware признаки `m3x_*`** (строятся в `honest_feature_builder` поверх сырых `ofz_auctions`): `m3x_cover`, `m3x_placement`, `m3x_yield_to_key`, `m3x_age`, `m3x_available`, `m3x_days_since`, `m3x_failed`.

## 3. Whitelist (что отобрано в LSI)

```python
M3_FEATURES = ["m3_auction_flag", "m3_Flag_Nedospros", "m3x_cover", "m3x_placement",
               "m3x_yield_to_key", "m3x_age", "m3x_available", "m3x_days_since", "m3x_failed"]
```

| Признак | Почему в whitelist |
|---------|--------------------|
| `m3x_cover` | event-time аномалия переподписки — низкий спрос = стресс |
| `m3x_placement` | доля размещения от предложения — Минфин не смог продать выпуск |
| `m3x_yield_to_key` | **премия доходности к ключевой** — цена, которую платит Минфин за спрос |
| `m3x_failed` | несостоявшийся аукцион (placed = 0) — сильный дискретный сигнал |
| `m3x_age`, `m3x_available`, `m3x_days_since` | **рецентность данных**: модель должна знать, насколько свежий сигнал аукциона она видит на дневной шкале |
| `m3_auction_flag`, `m3_Flag_Nedospros` | контекст: был ли аукцион и флаг недоспроса |

**Почему такой богатый набор (и почему это правильно).** На ревью обсуждалось «не слишком ли много фичей в M3». Решение — **оставить богатым**: ОФЗ важный и ёмкий канал, а разрежённость событий требует явных признаков свежести (`age`/`available`/`days_since`), иначе forward-fill «замораживал» бы старый сигнал без понимания его актуальности.

**Почему НЕ вошли:**
- `MAD_score_cover`, `MAD_score_yield_spread` (календарные) заменены на **event-aware** `m3x_*`: считать MAD по календарным дням для разрежённых аукционов некорректно (между аукционами ряд «стоит»). Event-time MAD считается **по событиям**, что устраняет искажение.
- `Flag_Perespros`, сырые объёмы — контекст/нестационарны.

## 4. Методика расчёта

### Агрегация аукционов за день
```python
offered = Σ offered_amount;  demand = Σ demand_amount;  placed = Σ placed_amount
cutoff_y = Σ(cutoff_yield · placed_amount) / placed        # доходность, взвеш. по размещению
cover     = demand / offered
placement = placed / offered
failed    = (placed == 0)
yield_to_key = cutoff_y − key_rate                          # премия к ключевой (as-of keyrate)
```

### Event-time MAD (ключевой приём для разрежённых данных)
Вместо календарного окна MAD считается по **последним событиям** в скользящем 3-летнем окне *среди самих аукционов*:

```python
для каждого аукциона i:
    w   = значения за (date_i − 3 года, date_i]    # только дни аукционов
    med = median(w);  mad = max(median(|w − med|), 0.05)
    score_i = (value_i − med) / mad
```
Затем **знак** настраивается под «вверх = стресс»:
- `m3x_cover`, `m3x_placement` → знак `−1` (низкий спрос/размещение = высокий стресс);
- `m3x_yield_to_key` → знак `+1` (высокая премия = высокий стресс).

Результат `merge_asof(direction="backward")` протягивается на дневной календарь.

### Признаки свежести
```python
age          = дней с последнего аукциона (cap 90)          # m3x_age
m3x_available = (0 ≤ age ≤ 10) ? 1 : 0                       # «есть свежие данные»
m3x_days_since = age, 0 до первого аукциона, cap 250         # m3x_days_since
m3x_failed     = (день входит в множество несостоявшихся)    # дискретный флаг
```
