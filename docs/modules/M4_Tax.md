# 📅 M4 — Налоговый календарь (Overlay)

## 1. Описание модуля

**Экономический смысл.** В единые дни уплаты налогов (НДС, НДПИ, налог на прибыль, страховые взносы) банковский сектор переводит крупные суммы со счетов клиентов в бюджет — рублёвая ликвидность временно **изымается из системы**. Это создаёт регулярное **календарное давление** на ликвидность вокруг 25-х чисел и концов кварталов.

**Ключевое отличие M4 от M1–M3, M5:** это **overlay**, а не модуль PCA. Налоговый календарь **детерминирован** — известен заранее на годы вперёд. Поэтому:

- M4 **не входит** ни в один whitelist и **не двигает** значение LSI;
- он отдаётся как **контекст** рядом с индексом (налоговая неделя / день, давление), чтобы аналитик мог интерпретировать всплеск LSI («это налоговый период» vs «это настоящий стресс»).

**За чем следим:** близость к налоговым срокам и интенсивность календарного давления — как фон для чтения LSI, а не как его вход.

---

## 2. Полный пул фичей (`m4_features`)

| Группа | Колонки | Смысл |
|--------|---------|-------|
| Флаги периодов | `Tax_Pre_Flag`, `Tax_Active_Flag`, `Tax_Post_Flag`, `Tax_Week_Flag`, `Tax_Day_Strict` | пред-/активный/пост-налоговый период, налоговая неделя, строгий налоговый день |
| Календарные границы | `is_quarter_end`, `is_year_end`, `is_month_end`, `Regime_Post_ENP` | концы периодов, режим после ЕНП (единого налогового платежа) |
| Давление | `tax_pressure`, `tax_pressure_smoothed`, `tax_proximity` | интенсивность давления, сглаженная версия, близость к сроку |
| Аномалии/сезонность | `MAD_tax_pressure`, `MAD_tax_proximity`, `Seasonal_Factor`, `Seasonal_Factor_raw` | MAD-аномалии давления/близости, сезонный фактор |

## 3. Whitelist (что отобрано в LSI)

```text
M4 — overlay: НЕТ признаков в whitelist (ни в GLOBAL_WHITELIST, ни в LOCAL_WHITELIST).
```

**Почему M4 исключён из индекса полностью:**

1. **Детерминированность.** Календарь известен заранее. Аномалия не может быть «неожиданной» — её нельзя трактовать как сигнал стресса без учителя.
2. **Ложная сезонная подсветка.** При включении M4 в PCA индекс регулярно «подсвечивал» каждый налоговый период как стресс, даже в спокойные времена — это раздувало false positives и смещало баланс вкладов.
3. **Роль контекста, а не драйвера.** Налоговое давление *усиливает* реальный стресс, но само по себе — ожидаемое, управляемое событие. Поэтому оно показывается рядом с LSI как объяснение, а не подмешивается в значение.

> На странице «M4 — Налоги» панель вклада в LSI явно показывает **0% / "вне PCA"** — это сделано намеренно, чтобы подчеркнуть overlay-природу модуля.

**Контекстное использование.** `Tax_Week_Flag`, `Tax_Day_Strict`, `MAD_tax_pressure` отдаются как метаданные (`tax_overlay`) вместе с ответом LSI — для интерпретации, не для скоринга.

## 4. Методика расчёта

### Источник
Парсится opendata-XML налогового календаря **ФНС** (`tax_calendar_downloader` → `parsers/tax_calendar`), из которого восстанавливаются даты налоговых обязательств.

### Флаги периодов
Вокруг каждой даты уплаты строятся календарные флаги:
- `Tax_Day_Strict` — строгий день уплаты;
- `Tax_Week_Flag` — налоговая неделя (окно вокруг срока);
- `Tax_Pre_Flag` / `Tax_Active_Flag` / `Tax_Post_Flag` — фазы относительно срока.

### Давление и близость
- `tax_proximity` — близость к ближайшему налоговому сроку (чем ближе, тем выше);
- `tax_pressure` — интенсивность ожидаемого изъятия (с учётом веса/сезонности срока), `tax_pressure_smoothed` — сглаженная;
- `MAD_tax_pressure` / `MAD_tax_proximity` — робастные MAD-аномалии этих рядов (методика MAD — см. [M1](M1_Reserves.md#4-методика-расчёта)), используются только как контекст.

> Код расчёта — `backend/src/services/m4_feature_builder.py`; источник — налоговый календарь ФНС, см. [`../data/README_DATA.md`](../data/README_DATA.md).
